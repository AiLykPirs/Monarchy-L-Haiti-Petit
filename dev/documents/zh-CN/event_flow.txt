每年事件流程与判定逻辑
========================

┌─────────────────────────────────────────────────────────┐
│              每年由 nextYear() 驱动                      │
├─────────────────────────────────────────────────────────┤
│  玩家点击"下一年 →" → nextYear() 被调用                  │
└─────────────────────────────────────────────────────────┘

~~~~~~~~~~~~~~~~~~~~~~~~
0. 前置检查
~~~~~~~~~~~~~~~~~~~~~~~~
  · 若还有角色未行动（非自动职业），弹出 confirm() 询问
    → 取消：本年不推进
    → 确认：标记 yearActionsDone，继续

───────────────────────────────────────────────────────────
第1步 ─ 年初初始化
───────────────────────────────────────────────────────────

  1a. 重置 _actedThisYear 标记（所有角色）
  1b. G.time += 1（年份推进）

  1c. processTargetFluctuations() ── 地图目标波动
      对所有非总部的地图目标，每个属性随机波动：
        tec/cul/prd/pop/mil/inf/tre/cal
      （使用目标自身的 _tec/_cul/… 掷骰表达式）

───────────────────────────────────────────────────────────
第2步 ─ 征服处理与效果到期
───────────────────────────────────────────────────────────

  2a. processConquests() ── 征服年度推进
      对每个活跃征服：
      · 若 playerMdt > oppMdt → 对方各属性 20% 概率 -d10
      · 若 G.time >= 开始年 + 持续年 → 结算胜负

  2b. 刷新行动面板标签

  2c. 压榨效果到期
      遍历 G._squeezeEffects：
        endYear 到达 → 从 aprd/apop 扣除对应值

  2d. 理政效果到期
      遍历 G._governEffects：
        endYear 到达 → 从 atec/acul 扣除对应值

  2e. 演兵效果到期
      遍历 G._drillEffects：
        endYear 到达 → 从 amil 扣除对应值

───────────────────────────────────────────────────────────
第3步 ─ 年龄增长
───────────────────────────────────────────────────────────

  3a. 所有活跃角色：c.age += 1
  3b. unavailableChars 角色：age+1
      若 age >= hel → 标记自然死亡，移入 deadChars
  3c. deadChars 角色：age+1
  3d. 人才市场待售角色：age+1
  3e. 特殊人才：age+1
  3f. generateTalentMarket() ── 刷新人才市场
      移除已购人才；10% 概率余下人才离开
      补充至 2 个名额；冷却到期则刷新特殊人才

───────────────────────────────────────────────────────────
第4步 ─ 悬赏到期
───────────────────────────────────────────────────────────

  4a. 每个悬赏：yearRemaining--
      若 ≤ 0 → 生成悬赏角色，加入 G.chars
      标记 _isTalent=true, _actedThisYear=true，记录日志

───────────────────────────────────────────────────────────
第5步 ─ 年度日志标题
───────────────────────────────────────────────────────────

  5a. addLog('=== 纪元 {G.time}年 ===')

───────────────────────────────────────────────────────────
第6步 ─ 全体角色年度检定
───────────────────────────────────────────────────────────

  对每个存活/非退场角色调用 overallCheck(c)：

    ├── 6a. grow(c) ── 年龄成长判定
    │    ┌─────────────┬──────────┬──────────────────────────┐
    │    │ 年龄段       │ 频率     │ 效果                      │
    │    ├─────────────┼──────────┼──────────────────────────┤
    │    │ ≤ 6 岁      │ 每2年    │ int+d4, cha+d4, sta+d4,   │
    │    │             │          │ sex+d4, psq+d4, hel±d4    │
    │    ├─────────────┼──────────┼──────────────────────────┤
    │    │ 7~21 岁     │ 每3年    │ int+d6, cha+d6, sta+d6,   │
    │    │             │          │ sex+d6, psq+d6, hel±d6    │
    │    ├─────────────┼──────────┼──────────────────────────┤
    │    │ 22~60 岁    │ 每5年    │ int+d8, cha+d8, sta+d4,   │
    │    │             │          │ sex+d4, psq+d4, con+d4,   │
    │    │             │          │ hel±d4                    │
    │    ├─────────────┼──────────┼──────────────────────────┤
    │    │ 61~80 岁    │ 每5年    │ sex-d4, con+d6, hel±d4   │
    │    │             │          │ 若 %10: int+d4, cha±d4,   │
    │    │             │          │ sta-d4                    │
    │    ├─────────────┼──────────┼──────────────────────────┤
    │    │ ≥ 81 岁     │ 每年    │ sex-d4, psq-d4            │
    │    │             │          │ 若 %5: int+d4, cha-d6,    │
    │    │             │          │ sta-d10, con+d6, hel±d4   │
    │    └─────────────┴──────────┴──────────────────────────┘

    ├── 6b. accident(c) ── 偶遇事件判定（仅 age%10==0）
    │    判定：compagainst(c.luc, c.cal)
    │    ┌────────┬──────────┬──────────────────────────────┐
    │    │ 结果   │ 标签      │ 效果                          │
    │    ├────────┼──────────┼──────────────────────────────┤
    │    │ 3      │ 牛逼      │ 2随机属性 +d8×3; luc-d4;     │
    │    │        │          │ cal+d8                       │
    │    │ 2      │ 大好运   │ 1随机属性 +d12; cal+d6        │
    │    │ 1      │ 好运     │ 1随机属性 +d6; luc+d4;       │
    │    │        │          │ cal+d4                       │
    │    │ 0      │ 未知     │ luc±d12; cal±d12             │
    │    │ -1     │ 厄运     │ 1随机属性 -d6; luc+d4;       │
    │    │        │          │ cal+d4                       │
    │    │ -2     │ 大厄运   │ 1随机属性 -d12; luc+d6       │
    │    │ -3     │ 傻逼     │ 2随机属性 -d8×3; luc+d8;    │
    │    │        │          │ cal-d4                       │
    │    └────────┴──────────┴──────────────────────────────┘
    │    随机属性池：int, cha, sta, sex, hel, wel

    ├── 6c. sta < 5 检查
    │    若 c.sta < 5 且存活 → hel -= d(6)，日志 [虚弱]

    └── 6d. deathCheck(c) ── 死亡判定
          若 c.age >= c.hel 且无 exitStatus →
          handleExit(c, 'natural')
          → 继承、储君继位、移入 deadChars，日志 [死亡]

───────────────────────────────────────────────────────────
第7步 ─ 人才互助
───────────────────────────────────────────────────────────

  7a. 找出 wel < 5 的存活 talent
      对每个：10% 概率最富 talent 转 d6 财富给他

───────────────────────────────────────────────────────────
第8步 ─ 重置额外累积值
───────────────────────────────────────────────────────────

  8a. etec, ecul, eprd, epop, emil, einf, eapo, emdt 归零

───────────────────────────────────────────────────────────
第9步 ─ 自动职业工作
───────────────────────────────────────────────────────────

  9a. 学者 ch(int)
      ┌────────┬──────────────────────────────────────────┐
      │ 结果   │ 效果                                      │
      ├────────┼──────────────────────────────────────────┤
      │ 失败   │ exp+1                                    │
      │ 成功   │ exp+1, int+1, wel+1                       │
      │ 大成功 │ exp+1, int+d4, wel+d4, edu+1, etec+1,    │
      │        │ 君主 int+1                               │
      │ 极限   │ exp+d4, int+d4+d4, wel+d6, edu+d4,       │
      │        │ etec+d4, 君主 int+d4, 宰相 int+d4        │
      └────────┴──────────────────────────────────────────┘
      基础：etec+=1, etre+=1

  9b. 艺人 ch(cha)
      基础：ecul+=1, einf+=d4
      ┌────────┬──────────────────────────────────────────┐
      │ 结果   │ 效果                                      │
      ├────────┼──────────────────────────────────────────┤
      │ 失败   │ exp+1, sta-1                              │
      │ 成功   │ exp+1, cha+1, wel+1                       │
      │ 大成功 │ exp+1, cha+d4, wel+d4, sta+d4, ecul+1,   │
      │        │ 君主 sex+1, 后宫 sex+1                   │
      │ 极限   │ exp+d4, cha+d8, wel+d6, sta+d6, ecul+d4, │
      │        │ 君主 sex+d4, 后宫 sex+d4                 │
      └────────┴──────────────────────────────────────────┘

  9c. 劳工 ch(sta)
      基础：eprd+=1, etre+=d4
      ┌────────┬──────────────────────────────────────────┐
      │ 结果   │ 效果                                      │
      ├────────┼──────────────────────────────────────────┤
      │ 失败   │ exp+1, sta-d4                             │
      │ 成功   │ exp+1, sta-d4, wel+d4, eprd+d4, etre+d4  │
      │ 大成功 │ exp+1, sta-d4-1, wel+d6, eprd+d6,        │
      │        │ etre+d6                                  │
      │ 极限   │ exp+d6, sta-1, hel-d4, wel+d6+d6,        │
      │        │ eprd+d6+d6, etre+d6+d6                   │
      └────────┴──────────────────────────────────────────┘

  9d. 士兵 ch(psq)
      基础：emil+=2
      ┌────────┬──────────────────────────────────────────┐
      │ 结果   │ 效果                                      │
      ├────────┼──────────────────────────────────────────┤
      │ 失败   │ exp+1, psq+1                              │
      │ 成功   │ exp+1, psq+1, con+1, emil+1, 君主 con+1  │
      │ 大成功 │ exp+1, psq+d4, con+d4, emil+d4,           │
      │        │ 君主 con+d4                              │
      │ 极限   │ exp+d4, psq+d6+d6, con+d10, emil+d6+d6,  │
      │        │ 君主 con+d6+d6                           │
      └────────┴──────────────────────────────────────────┘

───────────────────────────────────────────────────────────
第10步 ─ 重臣压榨
───────────────────────────────────────────────────────────

  10a. 宰相——从学者/艺人/无业者中选 top3 财富 + 1 随机
       每人 50% 被偷 2d4 财富；宰相 cal+d4, con+d8

  10b. 监工——从劳工/普侍/无业者中选（同上算法）
       监工 cal+d4, con+d8

  10c. 将军——从士兵/无业者中选（同上算法）
       将军 cal+d4, con+d8

───────────────────────────────────────────────────────────
第11步 ─ 皇帝唯一性检查
───────────────────────────────────────────────────────────

  11a. 若存在多于 1 个 profession=='皇帝' → 日志报错

───────────────────────────────────────────────────────────
第12步 ─ 政变判定
───────────────────────────────────────────────────────────

  12a. 识别豪强（>=3 项属性超过君主的角色）
       对比上一年名单，日志记录新增/移除

  12b. 魄力最高的豪强获日志"受到敬畏"

  12c. 若 coupYears % 5 == 0 → 全体豪强 con += d4

  12d. 若 coupCooldown <= 0 且豪强 >= 2：
       · 若君主变更且已有储君 → 政变被阻止，重置冷却
       · 否则 → runCoup()
         ┌──────────────────────────────────────────────┐
         │ runCoup() 流程：                              │
         │ 1. 豪强按魄力排序                            │
         │ 2. 两两 xch(con, con) 淘汰赛                 │
         │ 3. 冠军挑战君主：xch(冠军最高属性, 君主最高   │
         │    属性)，若 >= 2 → 君主被流放，冠军继位      │
         │ 4. mdtPenalty += 100，可能改朝换代            │
         └──────────────────────────────────────────────┘

───────────────────────────────────────────────────────────
第13步 ─ 婚约到期
───────────────────────────────────────────────────────────

  13a. 每个婚约：yearsLeft--
       若 == 0 → 成婚
       若涉及君主 → 更换正宫
       日志记录

───────────────────────────────────────────────────────────
第14步 ─ 随机配对
───────────────────────────────────────────────────────────

  14a. 从未婚、非皇室、非普侍、age>=12 的角色中
       随机选发起人，25% 找到异性对象
       若 ch(发起人最高属性) >= 1 → 成为恋人，订婚 d4 年

───────────────────────────────────────────────────────────
第15步 ─ NPC 生育
───────────────────────────────────────────────────────────

  15a. 从已婚非领导 NPC 中：10% 随机一对生育
       父母：hel-d4, sta-d4, sex-d10×3
       新生儿：各属性随机继承父母之一
       → 放入 unavailableChars
       → updateOrganization()

───────────────────────────────────────────────────────────
第16步 ─ 年度结算
───────────────────────────────────────────────────────────

  16a. cur.btre += cur.etre; cur.etre = 0
      （额外银库并入基础银库，额外银库归零）

───────────────────────────────────────────────────────────
第17步 ─ 普侍年度行动
───────────────────────────────────────────────────────────

  17a. processPushiAnnualActions()
       · 每个普侍：epop+=1, 君主 cal+=d10, eapo+=d5
       · 生育判定：
         ┌────────────────────────────────────────────┐
         │ 找异性对象（12~50岁，非君主/正宫/普侍，    │
         │ 不在冷却中）                               │
         │ 概率 = 25%~100%（取决于池大小和新奇度）     │
         │ 若尝试：ch(sexSum + tierMod)               │
         │ >= 1 → 生育成功                            │
         │ → 父母 hel/sta/sex 减少, cal/con 增加     │
         │ → 新生儿入 unavailableChars               │
         │ → 异性对象进入 d8 年生育冷却               │
         └────────────────────────────────────────────┘
       · 结束时 updateOrganization()

───────────────────────────────────────────────────────────
第18步 ─ 休养者处理
───────────────────────────────────────────────────────────

  18a. 每个休养者：若 btre >= 休养者数×4
       → 支付 d4 俸禄；君主 cal -= 1；mdtRestorerAccum += 1
       若资金不足 → 无人获俸禄

───────────────────────────────────────────────────────────
第19步 ─ 自动退职
───────────────────────────────────────────────────────────

  19a. processAutoRetirement()
       · 储君：_retireAccumulator += d8
       · 大臣：_retireAccumulator += d4
       · 对每个符合退职条件的角色（age>=18，非君主/正宫/
         休养者/普侍，非征服参与者）：
         ┌────────────────────────────────────────────┐
         │ 退职条件：wel<10 / sta<10 / 有在世子女 /   │
         │ 配偶已退 / age>80 / con<15                 │
         │ 第一年标记 → "有点累了"（无行动）          │
         │ 后续年份 → 累积概率，掷骰成功则退职        │
         └────────────────────────────────────────────┘

───────────────────────────────────────────────────────────
第20步 ─ 组织属性更新
───────────────────────────────────────────────────────────

  20a. updateOrganization() ── 完整重算
       ┌────────────────────────────────────────────────┐
       │ n = 存活人数 + 不可用人数                      │
       │ muln = 1 + n/10                               │
       │                                                │
       │ tec = emp.int + avg(int)×muln + etec + atec    │
       │ cul = emp.cha + avg(cha)×muln + ecul + acul    │
       │ prd = overseer.sta + avg(sta)×muln + eprd + aprd│
       │ pop = overseer.sex×0.5 + avg(sex)×0.5 + 2n     │
       │      + epop + apop                             │
       │ mil = (emp.psq+emp.con)×0.25                   │
       │      + (将军.psq+将军.con)×0.25                │
       │      + avg(psq)×muln + 士兵数                  │
       │      + emil + amil                             │
       │ apo = emp.cal + avg(cal)×(0.2+n/10)            │
       │      + eapo + aapo                             │
       │                                                │
       │ inf = 0.2×(tec+cul+prd+pop+mil)                │
       │      + 0.7×emp.con                             │
       │      + 0.125×重臣魄力和                        │
       │      + tre/7 + einf + ainf                     │
       │                                                │
       │ mdt = 500 - mdtPenalty + mdtRestorerAccum      │
       │      + inf - apo                               │
       └────────────────────────────────────────────────┘

───────────────────────────────────────────────────────────
第21步 ─ 游戏结束判定
───────────────────────────────────────────────────────────

  21a. checkGameOver()
       · 若 mdt < 0 → "天命已尽！"
       · 若无存活角色 → "树倒猢狲散！"
       · 若触发 → 标记结束，记录日志，返回（跳过后续）

───────────────────────────────────────────────────────────
第22步 ─ 重置并进入玩家行动阶段
───────────────────────────────────────────────────────────

  22a. yearActionsDone = false
  22b. actingCharId = null
  22c. beginYearActions()
       · 重置 _actedThisYear 标记
       · 自动职业（学者/艺人/劳工/士兵/普侍）标记为已行动
       · 更新 UI
  22d. addLog('新的纪元开始了。')
